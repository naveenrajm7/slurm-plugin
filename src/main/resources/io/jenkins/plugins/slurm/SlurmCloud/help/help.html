Configure Jenkins to use a SLURM cluster as a cloud provider for dynamically provisioning build agents.

This cloud provider will submit jobs to your SLURM cluster and use the allocated compute resources as Jenkins agents. Each agent runs as a separate SLURM job, allowing you to leverage the full power of your HPC infrastructure for CI/CD workloads.

Prerequisites:

  • SLURM cluster with sbatch, squeue, and scancel commands available
  • SSH access to SLURM compute nodes
  • Java installed on compute nodes
  • Network connectivity between Jenkins and SLURM nodes
