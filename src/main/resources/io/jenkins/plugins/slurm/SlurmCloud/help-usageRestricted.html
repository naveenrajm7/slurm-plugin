When enabled, this Slurm cloud will only be available to jobs in folders that have been explicitly granted permission to use it.

This provides fine-grained access control, allowing administrators to restrict which teams or projects can use specific Slurm clusters.

To grant permissions:

  1. Enable this option on the Slurm cloud
  2. Go to a folder's configuration
  3. Add the "Slurm" folder property
  4. Select which clouds are permitted for that folder

Permissions are inherited by subfolders, so you can configure access at any level of the folder hierarchy.

Note: When this option is disabled, the cloud is available to all jobs without restriction.
