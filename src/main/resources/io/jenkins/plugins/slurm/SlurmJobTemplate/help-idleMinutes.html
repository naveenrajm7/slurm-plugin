Idle Minutes controls how long the agent remains active after becoming idle (no builds assigned).

Behavior:

  • 0 minutes: Agent terminates immediately when idle (one-shot mode).
    ⚠️ Warning: May cause build assignment failures due to network delays.
    Use this only if you understand the risks.
  • 1 minute: Recommended minimum for run-once agents. Provides safety buffer for build assignment.
  • 5 minutes (default): Good balance - allows agent reuse while cleaning up promptly.
  • 10+ minutes: Keeps agents alive longer for high reusability across multiple builds.

Important Notes:

  • This setting always applies, regardless of "Run Once" setting
  • Agent does not terminate immediately on startup - it waits for work first
  • Idle timeout only starts after a build completes and agent becomes idle
  • Lower values = faster cleanup but less reusability
  • Higher values = more SLURM resource usage but better reusability

Recommended Settings:

Use Case Run Once Idle Minutes
One-shot agents ✓ Enabled 1-2 minutes
Reusable agents (low traffic) ✗ Disabled 5 minutes
Reusable agents (high traffic) ✗ Disabled 10-15 minutes

See IDLE_MINUTES_BEHAVIOR.md in the plugin repository for detailed documentation.
